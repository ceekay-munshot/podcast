# Project memory — Munshot Podcasts

## Document exports (PDF / Word) — generate files in JS, download as a Blob

**This app runs embedded in the chat.muns.io dashboard iframe**, sandboxed
`allow-scripts allow-same-origin allow-popups allow-forms allow-downloads` —
crucially **without `allow-modals`**. Per the HTML sandbox spec that makes every
script-initiated `window.print()` a **silent no-op** for the whole frame tree
(nested iframes inherit the sandbox; popups can't escape it without
`allow-popups-to-escape-sandbox`). File **downloads DO work** (`allow-downloads`).

**Method to always use for exports — do NOT rely on the browser:**

- **Generate the file client-side with a library and download it as a Blob**
  (`new Blob(...)` → `URL.createObjectURL` → a temporary `<a download>` click).
  This is the one delivery that can't be silently blocked in the iframe.
- **PDF:** build a real `.pdf` with **jsPDF** — `src/lib/pdfRender.ts`. Vector,
  selectable text; full house style drawn by hand (gradient cover painted to a
  `<canvas>` and embedded as an image, gold rules, drop-cap lead, idea cards,
  dark quote panel, zebra source table) with controlled pagination. Fonts use
  the **standard PDF families (Times / Helvetica / Courier)** — same serif/sans/
  mono split as the `.doc` — so nothing needs embedding. Content is a pure,
  testable **block model** (`weeklyBlocks` / `episodeBlocks`); rendering is
  isolated so generation can be unit-tested in Node (assert valid `%PDF` bytes).
  jsPDF is **lazy-imported** so it never weighs down the main bundle.
- **Word (.doc):** `src/lib/exportDoc.ts` + `exportWeekly.ts` — Word-compatible
  HTML downloaded as `application/msword`. Same Blob-download model.
- Lazy-import heavy assets on the download path (`./brandLogo` = base64 logo).

**Never** do these for exports here:
- ❌ `window.print()` / the browser's native "Save as PDF" (blocked in the iframe).
- ❌ Download an `.html` that defers to the print dialog when "PDF" is requested —
  the user wants an actual `.pdf` file, generated by a library.

## Verifying PDF output visually (dev only)

To eyeball generated PDFs headlessly: dev-install `@napi-rs/canvas` +
`pdfjs-dist`, polyfill `globalThis.document.createElement('canvas')` with napi
canvas (so the canvas-based cover renders), generate the PDF, then rasterize
pages to PNG with pdf.js. Remove those dev deps afterward — only `jspdf` ships.
